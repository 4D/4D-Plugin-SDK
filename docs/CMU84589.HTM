PA_LockDatabase

version 2003

PA_LockDatabase

Parameter Type Description
This command does not require any parameters

Description

The routine PA_LockDatabase locks the current database.

PA_LockDatabase

must be called before performing any log operation to ensure that no process will manipulate data (add record, modify, delete, etc.) while the plug-in uses the log file.

NOTE

A call to PA_LockDatabase must be "closed" by a call to PA_UnlockDatabase . If this is not done, the data base will stay locked.

Example

Lock-unlock the database. 

   PA_LockDatabase();
   /* . . . make log actions . . . */
   PA_UnlockDatabase();
